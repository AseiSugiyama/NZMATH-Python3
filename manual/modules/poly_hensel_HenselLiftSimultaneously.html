HenselLiftSimultaneously †

A class represents integer polynomials which will be lifted by Hensel's lemma.

REFERENCE:
G.E.Collins & M.J.Encarnación. Improved Techniques for factoring Univariate Polynomials, J.Symb.Comp. 21, 313--327 (1996).

INVARIANTS:

ai's, pi's and gi's are monic
f == g1*g2*...*gr (mod p)
f == d0 + d1*p + d2*p**2 +...+ dk*p**k
hi == g(i+1)*...*gr
1 == gi*si + hi*ti (mod p) (i = 1, 2,..., r)
deg(si) < deg(hi), deg(ti) < deg(gi)
p | q
f == l1*l2*...*lr (mod q/p)
f == a1*a2*...*ar (mod q)
ui == ai*yi + bi*zi (mod p) (i = 1, 2,..., r)

Constructor †

HenselLiftSimultaneously(target, factors, cofactors, bases, p) †

f = target, gi's in factors, hi's in cofactors and si's and ti's are in bases.

from_factors(target, factors, p [,ubound]) †

A class method to create and return an instance of HenselLiftSimultaneously, whose factors are lifted by HenselLiftMulti upto ubound (if it is smaller than sys.maxint, or upto sys.maxint).

The parameters satisfy that:

f == a1*a2 (mod p)

with a prime number p and ai's in factors.

Methods †

first_lift() †

Start lifting.

f == l1*l2*...*lr (mod p**2)

Initialize di's, ui's, yi's and zi's. Update ai's, bi's. Then, update q with p**2.

general_lift() †

Continue lifting.

f == a1*a2*...*ar (mod p*q)

Update ai's, bi's, ui's, yi's and zi's. Then, update q with p*q.

lift() †

The lift. You should call this method.
