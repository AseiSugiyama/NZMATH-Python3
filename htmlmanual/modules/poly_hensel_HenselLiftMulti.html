HenselLiftMulti †

A class represents integer polynomials which will be lifted by Hensel's lemma, so-called the quadratic method.

If the number of factors is two, you should use HenselLiftPair.

Target and all its factors are supposed to be monic.

Constructor †

HenselLiftMulti(f, factors, ladder, p, q) †

The parameters satisfy that:

  1. ai's of factors and f are all monic
  2. bi's of cofactors are product of aj's for i < j
  3. f == a1*...*ar (mod q)
  4. ladder is a tuple (sis, tis) and si's of sis and ti's of tis satisfy: ai*si + bi*ti == 1 (mod p), with positive integers p dividing q.

If p==q, q can be omit from the argument.

from_factors(f, factors, p) †

A class method to create and return an instance of HenselLiftMulti.

The parameters satisfy that:

f == a1*...*ar (mod p)
with a prime number p and ai's in factors.
(f, a1, a2, p)

Methods †

lift_factors() †

Update factors by lifted integer coefficient polynomials Ai's:

f == A1*A2*...*Ar (mod p*q) and
Ai == ai (mod q) (i = 1, 2,...,r),

then, q is updated by p*q.

PRECONDITIONS (automatically satisfied):

f == a1*a1*...*ar (mod q)
ai*si + bi*ti == 1 (mod p) and
p | q

lift_ladder() †

Update si's and ti's with Si's and Ti's:

ai*Si + ai*Ti == 1 (mod p**2) (i=1,2,...,r)
Si == si (mod p) (i = 1, 2,...,r)
Ti == ti (mod p) (i = 1, 2,...,r)

then, update p with p**2.

PRECONDITIONS (automatically satisfied):

ai+si + bi+ti == 1 (mod p)
